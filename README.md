# Time Recording Service

Сервис записывает текущую временную метку раз в секунду и сохраняет её в базе данных.
При временной недоступности базы данных сервис продолжает работу, буферизует записи
и автоматически дозаписывает их после восстановления соединения.

---

## Возможности

- Генерация временной метки раз в секунду
- Асинхронная запись в БД через рабочий поток
- Буферизация записей при недоступности базы данных
- Повторные попытки восстановления соединения с интервалом 5 секунд
- Сохранение буфера на диск при остановке приложения
- REST API для получения сохранённых записей в JSON
- Swagger-документация API
- Docker-ready запуск

---

## API

### GET /records

Возвращает сохранённые временные метки.

Параметры:

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|---------------|----------|
| page | int | 0 | Номер страницы |
| size | int | 100 | Размер страницы |

Ответ возвращается в порядке добавления записей в хранилище.
---
Порядок выдачи данных соответствует порядку вставки записей в хранилище.
